## 11:40 — 修复三市场诊断报告 JavaScript 语法错误

**问题**：部署到 GitHub Pages 后页面不显示股票卡片，只有表头和综合分析文字。

**根因**：`renderUS()` 函数中模板字面量内嵌的复杂三元表达式导致 JavaScript 解析失败：
```javascript
${s.lever?' ('+s.code.includes('SOXL')?'3x':s.code.includes('FNGD')?'-3x':'1.5xVIX')+')':''}
```
Node.js `-c` 语法检查通过但在浏览器环境中解析出错。

**修复方案**：
1. 添加辅助函数 `getLeverLabel(code)`, `getLeverBadge(lever, rating)`, `getLeverText(lever, rating)` 替代嵌套三元表达式
2. 为 `calcSummary()` 中的 Chart.js 调用添加 `try-catch`，防止 CDN 加载失败影响页面
3. 通过 GitHub API 直接更新文件（git push 因 token 失效卡住）

**验证结果**：
- cn-stocks: 9 只正常渲染 (5598 chars)
- hk-stocks: 11 只正常渲染 (6407 chars)
- us-stocks: 16 只正常渲染 (9681 chars)

## 12:37 — 修复三市场市值分布图表失调

**问题**：页面上"三市场市值分布"和"盈亏分布概览"两个 Chart.js 图表显示异常（空白/变形）。

**根因**：CSS Grid 布局中 canvas 元素没有固定高度的容器。Chart.js 设置 `responsive: true` + `maintainAspectRatio: false` 时，canvas 高度继承自 `.chart-box`（grid item），在 Grid 中计算结果异常——高度被拉伸到 **22655px / 22871px**，图表内容超出可视区域。

**修复方案**：
1. 给每个 canvas 包一层 `<div class="canvas-wrap">`，设置 `height: 260px; position: relative;`
2. 改为 `maintainAspectRatio: true`，利用 wrapper 固定高度约束
3. Chart.js CDN 从 cdnjs.cloudflare.com 更换为 cdn.jsdelivr.net（国内更快），并加 `onerror` 回退

**验证结果**：
- marketPie: 553×260 ✅（原 552×22655）
- pnlBar: 553×260 ✅（原 552×22871）
- canvas-wrap: 高度 260px 正常
- Chart.js 加载正常
